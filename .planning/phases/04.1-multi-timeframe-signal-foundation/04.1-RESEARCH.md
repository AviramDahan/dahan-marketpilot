# Phase 4.1 Research: Multi-Timeframe Signal Foundation

## Official QuantConnect Findings

- QuantConnect supports consolidating data from smaller bars into larger bars
  and using the consolidated output for algorithm logic.
- Time-period and calendar consolidator documentation both warn that dynamic
  universes must remove consolidators for removed securities to avoid compounding
  memory use.
- US equity market-hours documentation is the source for regular trading hours,
  holidays, early closes, late opens, and the exchange timezone.
- Time-modeling documentation distinguishes daily bar emission behavior and
  confirms that completed-bar timing must be explicit.
- Warm-up and manual-indicator documentation support explicit readiness checks;
  indicator readiness must be verified before use.

## MTF Design Implications

- Do not invent QuantConnect API names. Implementation must verify exact current
  LEAN APIs before coding source subscriptions, consolidators, calendar
  anchoring, warm-up, and cleanup.
- Prefer source subscriptions at the smallest required regular resolution that
  can safely build 1H and 4H completed bars.
- Exclude extended hours for v1 MTF signals unless a later documented decision
  changes this.
- Treat early closes and the 6.5-hour regular session mismatch as first-class
  test cases.
- Dynamic universe additions require indicator/consolidator registration;
  removals require cleanup.

## Recommended 4H Alignment

Use regular-session, market-open anchoring in `America/New_York`:

- full 4H bar: market open through open + 4 hours
- remaining same-day segment: partial-session bar
- default policy: partial-session bars are not signal-generating
- optional policy to test later: partial bars may support evidence only
- early-close days: if the session cannot produce a complete 4H bar, no 4H
  signal is generated for that symbol/session

Backtesting must compare this policy against other documented alignment policies
and a 2H alternative only if 4H proves technically unsafe or too sparse.

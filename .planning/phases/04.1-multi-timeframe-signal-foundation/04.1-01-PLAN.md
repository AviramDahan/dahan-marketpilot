# Plan 04.1-01: QuantConnect MTF Verification And StrategyMode Contract

## Objective

Create the central strategy-mode contract and verify the QuantConnect API surface
needed for multi-resolution signal foundations.

## Tasks

1. Add typed strategy-mode configuration, preferably in `config/strategy.yaml`,
   with exactly `daily_only`, `daily_filter_4h_setup`, and
   `daily_filter_4h_setup_1h_optional`.
2. Ensure `daily_only` is default, compatibility mode, and benchmark mode.
3. Keep strategy mode separate from environment modes `backtest`, `shadow`, and
   `paper`.
4. Fail closed on missing, empty, invalid, or unsupported strategy mode values.
5. Document and test QuantConnect source subscriptions, `TradeBar`
   consolidators, calendar/custom anchoring, RTH filtering, extended-hours
   exclusion, holiday/early-close handling, indicator warm-up, and dynamic
   universe registration/cleanup assumptions.

  exist after implementation

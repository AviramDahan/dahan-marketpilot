# Phase 4.1 Validation Contract

Phase 4.1 implementation must be deterministic and offline wherever practical.
It must not run live QuantConnect, cloud backtests, Paper Trading, Telegram
delivery, or broker flows.

## Required Test Coverage

- Strategy mode config accepts exactly `daily_only`,
  `daily_filter_4h_setup`, and `daily_filter_4h_setup_1h_optional`.
- Missing, empty, unsupported, or malformed strategy mode fails closed.
- Strategy mode is not confused with environment mode.
- `daily_only` preserves completed daily-bar behavior.
- MTF modes require Daily readiness before 4H setup evaluation.
- 4H setup readiness is mandatory in MTF modes.
- 1H confirmation is optional and cannot independently create or reject trades.
- Completed-bar models preserve timeframe, start, end, completion status,
  exchange timezone, regular-hours flag, partial-session flag, freshness, and
  source resolution.
- `SetupTiming` preserves strategy mode, signal timeframe, timestamp, bar
  start/end, completion status, timezone, session flags, freshness, source
  resolution, and later-valid-execution requirement.
- Tests prove no future bars, incomplete bars, or same-bar execution assumptions
  are used.
- Dynamic-universe consolidator registration and cleanup are represented by
  unit-testable contracts or documented integration tests.
- Early closes, holidays, DST, extended-hours exclusion, and partial-session
  behavior are represented by deterministic fixtures.

## Forbidden Behavior Tests

Static or contract tests must prove this phase adds no order creation,
position sizing, broker adapters, Paper Trading deployment, Telegram delivery,
real backtest execution, fake results, fake portfolio values, or live-trading
behavior.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MarketPilotScore outputs -> ranking | Scores can be sorted and compared, but they must remain audit labels only. |
| Multiple setup scores -> one symbol candidate | Duplicate setup candidates can create duplicate downstream entries if grouping is wrong. |
| Combined Swing readiness inputs -> gate output | Readiness criteria determine whether a combined strategy can be represented as available. |
| RankedCandidate -> future dashboard/reports | Audit output can be misread as trade instruction if unsafe fields leak in. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-11 | Tampering | `rank_candidates` grouping | mitigate | Tests prove one candidate per normalized symbol and supporting setup retention per D-13. |
| T-05-12 | Tampering | Ranking tie-break sort | mitigate | Tests prove D-14 order: score, confidence, risk quality, relative strength. |
| T-05-13 | Elevation of Privilege | `CombinedSwingReadiness` | mitigate | Gate remains disabled until all D-15 prerequisites are true and config allows activation; Phase 5 config keeps it disabled. |
| T-05-14 | Information Disclosure | Docs/tests/config | mitigate | Do not add credentials, tokens, account identifiers, or secret examples. |
| T-05-15 | Elevation of Privilege | `RankedCandidate` output | mitigate | Static and dataclass-field tests forbid entry, stop, target, quantity, order, broker, Paper, Live, Telegram, portfolio state, fake performance, and profitability fields. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | This plan installs no packages. If an executor needs a new package, stop and run the package legitimacy gate before any install. |

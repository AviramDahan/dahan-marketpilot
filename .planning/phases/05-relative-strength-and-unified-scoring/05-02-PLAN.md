<interfaces>
Scoring must consume setup evidence without mutating setup modules:

- Inputs are `SetupResult` objects and optional supporting evidence tuples, not orders, positions, holdings, or portfolio state.
- `NumericEvidence.name`, `NumericEvidence.value`, `NumericEvidence.threshold`, and `NumericEvidence.passed` are the component evidence contract.
- `SetupResult.rejection_reasons` is the source for hard rejection override.
- Scoring may define classification labels in `marketpilot/scoring.py`; setup modules must continue to have no classification or total score fields.
- Sector fit must be represented in scoring output as contextual evidence or an explicit not_evaluated/unavailable gate status until sector/industry source data is verified; it must not reject a candidate by itself in Phase 5 per D-04 and SCO-01.
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML scoring config -> scoring engine | Weights, thresholds, mappings, and gate policy drive classification output. |
| SetupResult evidence -> scoring engine | Setup evidence can be missing, invalid, stale, or rejected. |
| Scoring output -> ranking/reports | Classification labels are audit labels only and must not become order authority. |
| Later-phase gates -> scoring engine | Portfolio and activation gates are not implemented in this phase and must be explicit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | Tampering | `config/scoring.yaml` | mitigate | Loader rejects unsafe paper settings and score weights that do not total 100. |
| T-05-07 | Tampering | `score_setup_result` | mitigate | Required missing/invalid/stale scoring data creates fail-closed scoring failure and `REJECTED` classification. |
| T-05-08 | Spoofing | Classification gates | mitigate | Default portfolio and activation gates are not_evaluated/unavailable and cannot produce `BUY_CANDIDATE`; no fake gate pass values. |
| T-05-09 | Repudiation | `MarketPilotScore` output | mitigate | Preserve component scores, numeric evidence, confidence inputs, hard rejection reasons, and classification reasons. |
| T-05-10 | Elevation of Privilege | Classification labels | mitigate | Tests and docs state labels are audit-only; no orders, sizing, portfolio mutation, entry/stop/target/order intent, Telegram, Paper/Live deployment, credentials, or fake performance. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | This plan installs no packages. If an executor needs a new package, stop and run the package legitimacy gate before any install. |
</threat_model>

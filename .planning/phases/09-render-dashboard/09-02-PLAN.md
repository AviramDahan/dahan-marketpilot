<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Streamlit auth form | Untrusted password input crosses into server-side comparison. |
| Render env/config -> auth helper | Secret values are loaded at runtime and must not leak into repository, UI, or diagnostics. |
| Streamlit session -> dashboard data | Authenticated session state controls whether dashboard data is fetched/rendered. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-02-01 | Spoofing | dashboard/auth.py | mitigate | Use single env-var password, constant-time comparison, and authenticated session flag only per D-11/D-12. |
| T-09-02-02 | Information Disclosure | dashboard/app.py | mitigate | Gate all data fetch/render paths behind auth; tests assert no dashboard data before login. |
| T-09-02-03 | Tampering/Elevation | dashboard/app.py | mitigate | Static/read-only tests reject order, Telegram send, Paper mode switch, recovery approval, export, and mutation controls per D-20/D-21/D-22. |
| T-09-02-04 | Information Disclosure | dashboard/config.py | mitigate | Config contains env var names only; safe diagnostics redact password-like values. |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | No dependency change is planned in 09-02. If Streamlit dependency declaration is changed here, stop and use the 09-04 package legitimacy checkpoint first. |
</threat_model>

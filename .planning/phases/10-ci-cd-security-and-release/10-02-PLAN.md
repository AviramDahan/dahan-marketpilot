## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repository docs/tests -> operator | Security review and testing docs guide release decisions. |
| Workflow files -> GitHub Actions runtime | Security tests inspect CI/CD behavior before workflows run. |
| Dashboard health -> Render dashboard | Security review confirms health checks remain read-only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Information Disclosure | Workflows and docs | mitigate | Static tests reject literal secret values, env dumps, and unmasked external URLs/tokens. |
| T-10-02-02 | Elevation of Privilege | Dashboard and workflow actions | mitigate | Security review and tests confirm dashboard actions stay view/refresh/login/logout and workflow permissions stay least-privilege. |
| T-10-02-03 | Tampering | Trading safety boundaries | mitigate | Tests and review reject real-money, real broker, Paper/live deploy, and dashboard mutation behavior. |
| T-10-02-04 | Repudiation | Security evidence | mitigate | `SECURITY_REVIEW.md` uses explicit `passed`, `failed`, `skipped`, and `not_run` statuses. |
| T-10-02-SC | Tampering | Package/action review records | mitigate | Security review records the 10-01 action SHA and Lean package legitimacy checkpoint outcome. |
</threat_model>

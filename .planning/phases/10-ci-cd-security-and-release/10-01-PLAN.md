<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub runner -> repository | Workflow code executes repo-controlled commands on hosted runners. |
| GitHub runner -> GitHub Secrets | Optional QuantConnect and dashboard checks read secret names/values from GitHub Actions. |
| GitHub runner -> QuantConnect Cloud | Guarded workflow may sync or request cloud backtests only after operator confirmation. |
| GitHub runner -> Render dashboard | Health workflow may perform read-only GET checks against the dashboard URL. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01-01 | Information Disclosure | `.github/workflows/*.yml` | mitigate | Keep default CI secret-free; mask non-empty external secrets; never echo URLs/tokens; static tests reject secret use in `tests.yml`. |
| T-10-01-02 | Tampering | `quantconnect.yml` | mitigate | Manual dispatch, operator confirmation, branch/secret guards, no Paper/live deployment commands, and explicit `not_run` when prerequisites are absent. |
| T-10-01-03 | Elevation of Privilege | Workflow token permissions | mitigate | Set `permissions: contents: read` and test against broad permission grants. |
| T-10-01-04 | Repudiation | External workflow outcomes | mitigate | Write `passed`, `skipped`, or `not_run` evidence to step summary; never convert missing external setup into success. |
| T-10-01-SC | Tampering | GitHub actions and Lean install | mitigate | Blocking human checkpoint for action full SHA pins and Lean PyPI package legitimacy before workflow install/use. |
</threat_model>
